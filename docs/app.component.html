ngx-pops

Dynamic Components Library

ngx-pops on GitHub

Rendering dynamic components is extremely easy using pops. You can render any component you want, using any data structure you want - as long as it inherits PopComponent, pops will handle component composition and destruction. You can use multiple containers in conjuction with each other by labeling them. Below are a couple of examples of how to use the library.

A perfect use case for pops is notifications. Create your notification component, pass it to doPop() along with any payload you might want to display, and pops will render them for you inside the container component.

ms

If you are looking for something less serious, try these balloons. Pops will pop them automatically after 3000 ms. You can blow up one, blow up ten, pop them manually or pop them all at once. You can also blow up an eternal balloon.

Charts (Demo powered by charts.js)

Creating a dashboard is also easy. Create your charts (try charts.js / ng2-charts), pass the data you want to display, and boom! You could display dynamic data using observables. This example just uses interal observables - you can compose your charts in whatever way you want.
